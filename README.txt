API_Web_Github

•	Une API est un ensemble de services offert par un logiciel à d'autres logiciels. Grâce aux API, les programmes informatiques peuvent interagir entre eux.
•	 On appelle API web une API accessible via les technologies du Web, notamment le protocole HTTP ou sa version sécurisée HTTPS. Un nombre croissant de sites et de services en ligne proposent des API web destinées aux développeurs. Elles offrent à ces développeurs le moyen d'exploiter leurs données et leurs services. Elles utilisent souvent le format JSON. 
•	On consomme une API web dans un programme au moyen d'un appel AJAX. Auparavant, il faut étudier sa documentation ou la tester à l'aide d'une extension de navigateur comme RESTClient ou Postman. 
•	De plus en plus de sites et de services en ligne proposent des API web. Certaines sont ouvertes tandis que d'autres sont soumises à authentification, par exemple au moyen d'une clé d'accès. 


L’objectif du projet est d'utiliser l'API web ouverte de GitHub pour afficher quelques informations sur le profil d'un développeur ou d'une organisation. 
Sa documentation est disponible à cette adresse.
L'API web ouverte de GitHub est limitée à 60 appels par heure. Vous pouvez consulter votre utilisation actuelle à l'URL https://api.github.com/rate_limit.
